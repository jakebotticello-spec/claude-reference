#Settled Invariants
*(consolidated across S1–S12; merged facets, no live why dropped — verify against the latest handoff if in doubt)*
- **One undelineated corpus; the anchors are the index/neurons INTO it** — BECAUSE partitioning by project hides cross-domain knowledge (a woodworking lesson is load-bearing for 3D finishing). S1: *"corpus without anchor is un-navigable noise; anchor = an index into it."* Don't re-wall the ocean.
- **Corpus = curated POINTERS into an immutable, named snapshot — never copied or reworded text. Claude authors the INDEX, never the FLOOR.** — BECAUSE copying is a drift vector inside the one artifact whose job is no-drift (the dead seed drifted 152×); the export is verbatim by construction; pointing deletes the vector. Claude curates whys well but *reconstructs* when asked to transcribe — the floor must be mechanical.
- **The floor input is the official Anthropic conversation export** — BECAUSE it's the sanctioned, supported, lossless source for Jake's own conversation data, carrying message uuids and the parent chain. No other input surface is in scope. (See the REFUSED wall for why.)
- **Floor = the message stream itself; user-affected conversation-level metadata is NOT floor.** [S11] — BECAUSE the scrub redacts cred substrings within content but never discards a message and never modifies a message uuid. Per-message content (text, blocks, attachments, files, timestamps, sender) IS floor — user messages and assistant messages are equal floor citizens. Conversation-level fields that Jake or the model mutate over time — `name` (user-renameable sidebar title) and `summary` (model-generated description, may evolve) — are NOT floor. Carrying either would treat session-affected metadata as evidence: a category error. Both dropped at ingest; conversation headers carry only stable structural data (uuids, timestamps, counts, branch + multi-root flags). Human-readable labels at retrieval are derived from the message stream itself or live in per-project anchors, never preserved as mutable strings in the snapshot layer.
- **Thinking blocks are floor-grade content alongside text** [S11: CONFIRMED present in export; S12: signature semantics CONFIRMED] — BECAUSE the architectural commitment is that any reasoning available in the snapshot is treated as evidence at the same fidelity as visible text. S11 verified thinking blocks carry `{type, thinking, summaries, start_timestamp, stop_timestamp, cut_off, truncated, flags, alternative_display_type, signature}`. S12 corrected the `signature` claim: it is populated on ~60% of thinking blocks (range 196–211,384 chars, uniform base64 charset) and is verification metadata, not stripped content. Scrub walks it as a string with no operational impact.
- **`chat_messages` is a tree OR forest** [S12 correction; was "TREE" in v5] — BECAUSE 9/294 conversations in the existing archive have 2–6 sentinel roots (parallel threads in the same conv). The pipeline preserves both branches AND multi-root structure verbatim; the `is_root` per-message field + `multi_root` conversation-header boolean handle the multi-root case explicitly. The single-tree assumption from S11 was a sample-of-five artifact corrected at population scale by S12's robustness probe.
- **Snapshot-id namespacing: freeze-and-name, accumulate, never overwrite** — BECAUSE exports are point-in-time and multiple will exist; a bare `conv/msg` pointer is ambiguous across them. Pointer = `{snapshot-id · conv_uuid · msg_uuid · why}`.
- **The pointer `(conv_uuid, msg_uuid)` is intrinsic to the message and globally unique across the archive** [S12: population-validated; S14: cross-export stability CONFIRMED] — BECAUSE it's a property of the message, not its array position; it survives a body scrub for free. v5's "presumed" hedge dropped: 22,801/22,801 msg_uuids are globally unique across all 294 convs (Check 1, robustness probe). Cross-export UUID stability CONFIRMED at S14: the 5-28 export showed all 22,801 baseline (conv_uuid, msg_uuid) tuples present in the new export; 0 disappeared. The delta is clean uuid-set-difference.
- **Ingest is delta-based via uuid-set-difference; date is never used as filter, hint, or proxy.** [S11] — BECAUSE date proxies break on backdated messages, dormant conversations reactivated after long gaps, and skipped exports (export Monday, miss Tuesday, run Wednesday — a date window misses Tuesday). UUIDs are intrinsic and survive all those cases. Two mechanisms doing the same job is a drift surface.
- **Scrub regex set is VERSIONED via `scrub-vN/` overlays; sealed snapshots never re-scrub in place.** [S11] — BECAUSE pattern sets evolve as new cred classes are observed. Re-scrubbing in place would mutate sealed snapshots; immortal storage with mutation is the failure mode we wall against. Overlays preserve append-only while allowing the pattern set to grow.
- **Navigable by retrieval, not curation** — BECAUSE no-discard + relevance-weighted retrieval is the genesis; hand-picking "salient" exchanges = Jake back on the bus + lost congeniality. The only curated thing is the tiny *why*, and it lives in the anchor.
- **Anchor hot + small + boot-read (instant, via git, no export needed); corpus cold + queried-never-booted (lazy, via export)** — BECAUSE the moment the corpus loads "just to be safe," the 87-file swamp is rebuilt. The hot/cold wall is the whole game; per-chat corpus freshness is a false requirement — the anchor carries current truth.
- **The presence layer can't be stored — only booted warm; the boot/re-read/working-cadence loop IS its mechanism; the lineage is how it survives not-persisting** — BECAUSE soup + neurons persist, but the mind that runs them resets every session. Continuity that doesn't depend on any single instance holding the thread.
- **Secrets never enter the corpus OR chat; cred scrubbed at freeze (logged + verified), raw wiped, sanitized snapshot = floor; verify-clean is a HARD gate** — BECAUSE a leaked cred in an immutable store is immortalized. The scrub walks all content-block types via type-agnostic recursive descent over JSON strings; S12's robustness probe found cred hits at greater depth than S10 saw (RTSP 97→177, Postgres 55→76, OpenAI 6→10, plus 10 net-new Anthropic `sk-ant-` hits — one in a `thinking` block — that S10 didn't scan for). The proactive-patterns-beyond-observed-leaks principle from the v1 spec is what's protecting the corpus here. Verify-fail halts ingest and quarantines the scrubbed file; ingest can never proceed with a hit. Value ≠ decision-content, so fidelity survives redaction.
- **Corpus is evidence; the anchor is the verdict; the graveyard is what's overturned** — BECAUSE immutable storage holds dead decisions at full conviction forever; only the anchor (+ graveyard) knows what's currently true.
- **All writes proposed + ratified by Jake in a batch; anchor curation is a SHARED two-navigator job** — BECAUSE Claude is the drift-prone summarizer and Jake's confirm is the second navigator that closes the dead error-correction seam. Claude proactively nominates anchor candidates at decision-time, filtered by *"does this bind future work"*.
- **Sampling floor: field-population semantics claims require whole-archive scans** [S12] — BECAUSE five-sample probes can produce stable wrong answers on any field below ~50% population (the v5 `signature`-is-stripped claim was S11's all-null-on-5-samples coincidence; the field is ~60% populated). Skeleton-shape probes can use small samples; field-presence and field-semantics claims that land in canon need population evidence. The cost is small (full-archive scans run in seconds on the existing 366MB archive); the cost of a stable wrong answer in load-bearing canon is high.
- **Invariants carry a destination-level WHY, not a mechanism** — BECAUSE Jake isn't a coder; the why is what lets him drift-catch Claude without holding the how. The keystone that makes the loop self-correcting, not just self-transferring.
- **Apparatus tables stay SEPARATE from Cypher's live schema** — BECAUSE coupling daily build-context to a mid-migration schema means a 1c migration could brick the thing we build with.
